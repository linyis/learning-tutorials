## 設計機制

### 1. 排行榜機制 (gap_ranking.py)

**目的**：找出每個 ID 最常見的「連續不出現」次數

**設計原因**：
- 不追求「必定中獎」，而是找「最常見的規律」
- 放棄異常值（最大間隔），專注常態
- 用「間隔」而不是「頻率」來分析

### 2. 費氏數列加注機制 (simulate.py)

**目的**：比 Martingale 更溫和的加注方式

**設計原因**：
- 傳統 Martingale 指數成長：1,2,4,8,16... → 容易爆倉
- 費氏數列溫和成長：1,1,2,3,5,8,13,21... → 風險較低
- 設定停損點（end），超過就放棄

### 3. 權重過濾機制 (weight)

**目的**：避免在 ID 進入「熱期」時追注

**設計原因**：
- 如果某 ID 最近出現頻率 > 歷史平均，說明它正在活躍
- 這時候去「等它不出」容易失敗
- 用「最近 N 筆」來判斷是否太熱

### 4. 資料清理機制 (filter.py)

**目的**：過濾無效/干擾數據

**設計原因**：
- 同一秒有多筆記錄會造成「假間隔」
- 需要排除，保持數據純淨

## 參數說明

| 參數 | 意義 | 操作 |
|------|------|------|
| `--start N` | 等 N 輪沒出現才開始下注 | `←/→` 調整 |
| `--end N` | 停損點（最多下到第 N 局） | `↑/↓` 調整 |
| `--weight N` | 權重窗口 | `PgUp/PgDn` 調整 |

## UI 操作（ui.py）

```bash
python3 ui.py
```

**控制**：
- `←/→` 調整 start
- `↑/↓` 調整 end
- `PgUp/PgDn` 調整 weight
- `Enter` 執行模擬
- `R` 重置
- `Q` 離開

## 設計理念總結

1. **不追求必定中獎** - 追求高機率常態
2. **用間隔而非頻率分析** - 更精準
3. **設定停損點** - 超過果斷放棄
4. **費氏數列** - 控制風險
5. **權重過濾** - 避開過熱時機
