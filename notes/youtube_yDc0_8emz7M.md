# 影片內容分析：Agent Skill 核心概念解說

## 1. 核心概念
影片主要介紹了由 Anthropic 推出並後續開放為標準的 **Agent Skill** 技術。其核心概念與機制如下：
*   **定義**：Agent Skill 本質上是大模型可以隨時翻閱的「說明文檔」（`skill.md`），用於定義特定任務的行為規範、輸出格式及執行規則。
*   **開放標準**：從最初僅用於提升 Claude 表現，演變為支持跨平臺、跨產品複用的通用設計模式（VS Code、Cursor 等均已支持）。
*   **按需加載機制**：為了節省 Token 資源，系統初始僅向模型發送 Skill 的「元數據」（名稱與描述）；只有當模型判斷需要該 Skill 時，才會加載完整的指令內容。
*   **高級功能**：包含 **Reference**（條件觸發加載外部文檔）與 **Script**（條件觸發執行代碼），實現更深層的自動化與上下文管理。

## 2. 舉例說明
影片中通過一個「會議總結助手」的實作過程，演示了 Agent Skill 的三層用法：
*   **基礎用法**：創建一個 `skill.md`，規定會議總結必須包含「參會人員、議題、決定」三個格式，模型讀取後即按此格式輸出。
*   **高級用法（Reference）**：引入「集團財務手冊」作為參考文檔。設定規則僅當會議內容涉及「錢、預算、採購」時，才加載該手冊並檢查合規性（如住宿補貼是否超標），否則不加載以節省資源。
*   **高級用法（Script）**：創建 `upload.py` 腳本。設定規則當用戶提到「上傳、同步」時，自動執行該腳本將總結內容發送到服務器，實現行動自動化。

## 3. 應用場景
根據影片內容，Agent Skill 可應用於以下場景：
*   **智能客服**：規範回應語氣（如先安撫情緒）、限制承諾權限，確保服務標準化。
*   **會議管理**：自動生成結構化總結，並根據內容動態關聯財務合規或法務風險提示。
*   **自動化工作流**：根據自然語言指令觸發代碼執行，如文件上傳、數據同步等任務。
*   **知識庫動態檢索**：避免將所有規章制度一次性輸入模型，而是根據話題動態加載相關政策文檔。

## 4. 與其他概念的關聯
*   **MCP (Model Context Protocol)**：影片明確提到將 Agent Skill 與 MCP 進行比較，兩者均屬於 AI Agent 領域中解決模型與外部資源交互的相關協議或設計模式。
*   **AI Agent 設計模式**：Agent Skill 被定義為 AI Agent 領域的一個「通用設計模式」，超越了單一產品（如 Claude）的範疇。
*   **Prompt Engineering**：本質上是結構化、持久化且可模塊化管理的系統提示詞（System Prompt）進化版。

## 5. 簡短總結
Agent Skill 是一種讓大模型能夠按需讀取指令文檔、參考資料甚至執行代碼的開放標準，有效解決了傳統提示詞重複粘貼與上下文資源浪費的痛點。透過「元數據檢索」與「條件觸發加載」機制，它在保持靈活性的同時大幅優化了 Token 使用效率。這標誌著 AI Agent 開發正從單一定製走向模塊化、標準化的新階段。
