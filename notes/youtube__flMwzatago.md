## AI 影片分析

這部影片帶領觀眾從零開始認識並實際搭建 AI Agent，以目前流行的 N8N 自動化工具為平台，逐步解析什麼是 Agent、它與傳統自動化工作流的差異、核心組成元件，以及如何組裝一個能夠每日發送新聞早報並根據日程提供建議的個人化 Agent。影片作者強調，Agent 就像一個能夠思考、記憶和執行任務的數位員工，與固定流程的自動化工作流不同之處在於它具備動態推理和自主決策的能力，能夠根據情境選擇合適的工具並調整執行策略。整個教學過程從基本概念講解到實際操作，涵蓋了 API 與 HTTP 請求的基礎知識、N8N 平台的安裝與操作，以及飛書多維表格、日曆等工具的串接方法。

影片首先釐清了一個常見的混淆點：自動化工作流並不等於 Agent。傳統的自動化工作流雖然可以串接多個步驟、使用大模型甚至連接各種工具，但它遵循預先定義好的固定規則，從 A 到 B 再到 C 的線性執行過程中沒有任何推理或自主決策。相比之下，即使是最簡單的 Agent，在被問及「今天有哪些新聞」時，也會主動察覺自己需要獲取新聞數據，隨後調用飛書多維表格查詢最新內容並進行總結回覆。這種根據任務需求動態規劃並選擇適當工具的能力，正是 Agent 的核心特質。

要實現上述能力，Agent 需要三個關鍵組件协同運作。第一個是「大腦」，也就是提供智能推理能力的大語言模型，如 Claude、GPT-4 或 DeepSeek 等，它們負責理解任務、進行規劃和生成語言回覆。第二個是「記憶」，它讓 Agent 能夠記住之前的對話脈絡和執行過程中的操作紀錄，並利用這些上下文資訊來優化決策，記憶的形式可以是最近幾輪對話、從文檔或資料庫中提取的外部資訊。第三個是「工具」，這是 Agent 與外部世界互動的主要方式，包括主動獲取資訊（如搜尋引擎、資料庫查詢）、採取行動（如發送郵件、創建日曆行程、執行代碼），以及編排其他 Agent 或工作流。影片特別強調，即使是目前最複雜的 Agent 系統，其本質仍然是由這三個基礎元件組合而成。

影片進一步說明了 API 和 HTTP 請求這兩個實用概念的區別。API（應用程式接口）就像 ATM 機上規定好的按鍵選項，定義了你可以發送什麼樣的請求以及獲得什麼樣的結果；而 HTTP 請求則是你實際發送的那條訊息，通常以 JSON 格式組織數據。理解這個區別對於後續自定義工具串接至關重要。N8N 作為一個開源的視覺化工作流與 Agent 搭建平台，被形容為「AI 自動化的瑞士軍刀」，其核心運作方式是通過拖放不同功能的節點來組裝流程，每個節點代表一個特定的操作或步驟。平台內建的 AI Agent 節點已經整合了大腦、記憶和工具三大組件，讓使用者能夠快速構建具有推理能力的 Agent。

在實際操作環節，影片展示了如何一步步搭建一個新聞早報 Agent。這個 Agent 會每天定時運行，先後查詢飛書多維表格中近兩天的新聞資料，以及用戶的日曆行程，然後根據新聞內容和日程安排生成一份客製化的早報，如果發現用戶有空閒時間且沒有重要新聞，還會推薦電影或戶外活動。操作過程中，作者詳細演示了如何配置 DeepSeek 大模型、設置記憶窗口、串接飛書多維表格和日曆的 API，以及如何編寫有效的提示詞來定義 Agent 的角色、任務、可使用的工具和輸出格式。提示詞的撰寫被認為是整個過程中最關鍵的環節之一，作者建議直接請 AI 幫忙生成結構完整的提示詞，包含角色定義、任務概述、可用工具清單、行為約束和輸出規範等要素。

影片最後強調了幾個重要的實用原則：能用提示詞解決的問題就不要用工作流，能用工作流實現的功能就不需要 Agent，單一 Agent 能完成的任務就不必構建多 Agent 系統，應該盡量保持架構的簡潔性。此外，作者也提醒觀眾設置「護欄」或限制條件的重要性，特別是當 Agent 需要提供給他人使用或作為企業服務時，適當的約束可以防止 Agent 產生幻覺、陷入循環或被惡意利用。對於想要進一步探索的觀眾，作者建議可以深入研究 N8N 的進階功能、多 Agent 系統架構、各類免費 API 的串接方法，以及如何根據個人需求擴展 Agent 的工具庫和工作流程。
